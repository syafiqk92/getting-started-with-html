The basics of the web and HTML

The web

The web, also known as www (which stands for world wide web) is a collection of HTML documents that make up the internet as we know today. It is linked together by hyperlinks.

Major components of the web

The webs are made up of the following components:

  • Browser
  • Internet
  • Servers

In simple terms, this is how internet works. Users use browser to access files on the server. However, user cannot access the server directly. Browser needs to send request to the server and the server will respond to the request by displaying the file on the browser. The request are made using a protocol called HTTP.

Browsers are software applications that users use to access information on the server. Examples of browsers include Safari, Google Chrome and others.

Servers, on the other hands are computers (pretty much like my computer) that have specialised function of hosting files.

The internet is simply a network of computers that perform a wide array of functions related to retrieving and storing information (although the list of functions that internet can do includes more than this).

Now that we understand the basics of how internet works, let's move on to more details information on HTML.

HTML

HTML stands for Hyper Text Markup Language. The HTML pages is the core document in the web although the web can include literally all types of files. HTML contains text, markup, references to other documents and links to other pages.

HTML is a markup language. Markup language is a system in which a document is annotated in such a way to allow browser to distinguish text from syntax. HTML has specific rules in annotating the text and the principle of that is by using tags.

HTML gives instructions to web browser on how to display the text.

HTML markup is made up of tags. The general form of tags is as follow. This whole thing can be referred to as elements.

<name>content</name>

Generally, all tags require opening and closing tags. An exception to this rule is void tag. Here is an example of a void tag

<img>

Next, we are going to explore on a couple of tags.

The <b> tag

This makes the text bold. The form is as follow:

<b>this text is bold</b>

Result: this text is bold

The <em> tag

This makes the text italics. The form is as follow:

<em>this text is italics</em>

Result: this text is italics

Note that any single typos can lead to big problems as computers follow instructions literally.

HTML attributes

Some tags can have attributes associated with them. The general form is as follow:

<tag attr="value">this text has tag with attributes</tag>

Some tags may have more than one attribute. One example of a tag with attribute is the anchor tag.

The general form of an anchor tag is as follow:

<a href="ww.google.co.uk">this text is anchored to the link and will take you to google</a>

Result: this text is anchored to the link and will take you to google

<img> is an example of a void tag. Void tag do not require a closing tag. The form is as follow:

In-line vs block

Some tags are in-line while others are block. Tags that are block makes an invisible box around the content whereas in-line just create a new line (in the case of break tag). Simply, in-line tags do not make an invisible block.

Container elements

<span> is in-line whereas <div> is block. Their function is to contain the content and they may have attributes (this is useful when adding class attributes in CSS).

The structure of HTML documents

This is how a complete HTML documents look like.

<!DOCTYPE HTML> <html>
<head>
   <title>title</title>
</head>
<body>
   <b>content</b>
</body>
</html>

Doctype is a tag that defines the HTML. HTML5 has a nice clean doctype. <HTML> tag surrounds the entire document. The <head> tag defines the head and it may contain meta data, java script and CSS source code. The body tag is the content of the document.
